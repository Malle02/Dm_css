# Projet CSS : 

## Démarche suivie

Pour ce projet, j'ai utilisé une approche basée sur les Flexbox pour gérer la mise en page de la majorité des éléments. Cela m'a permis de répartir efficacement l'espace entre les sections, tout en garantissant une compatibilité avec les tailles d'écran mobiles grâce à des règles CSS `@media`.



### 1. Navigation
La barre de navigation utilise Flexbox pour disposer les liens de manière bien aligné. Les sous-menus apparaissent au survol des liens principeaux et sont également alignés avec la partie principale (`<main>`) et la colonne latérale (`<aside>`),comme exigé.

### 2. Articles et contenu
Les articles sont disposés en colonnes égales avec Flexbox. Chaque article est conçu pour être adaptable en fonction de la largeur de la page.

### 4. Responsivité
J'ai implémenté des points de rupture (`@media`) pour garantir que la page soit responsive. Le layout passe d'une disposition en colonnes sur desktop à une disposition en ligne sur mobile.

## Pourquoi je n'ai pas utilisé Bootstrap

Je n'ai pas utilisé Bootstrap pour ce projet, car je ne me suis pas mis sur le projet un peut en retard ce qui ma empêcher de l'implémenter de manière correcte sans affecter le style que j'avais déjà établi. J'ai rencontré des difficultés personnelles ces dernières semaines qui ont limité le temps que j'ai pu consacrer à ce projet. J'ai donc fait le choix de prioriser la fonctionnalité et le design en utilisant uniquement CSS. Cependant, je comprends  que je n'ai paas ete a la hauter pour ce projet mais je donnerais le meilleiur de moi meme pour les futurs projets.

## Problèmes rencontrés et améliorations possibles

Le principal défi a été de finaliser le projet à temps. Comme mentionné, j'ai dû terminer certaines parties du projet dans la précipitation. Si j'avais consacré  plus de temps, j'aurais optimisé l'utilisation de Bootstrap et amélioré encore plus la gestion du responsive design.
